Architecture

The application is an AWS-first static + serverless design.

Layer AWS Service Responsibility
Edge CloudFront Serve static UI, route /api/* and /files/* to Lambda Function URL, apply route rewrites.
Frontend S3 (private + OAC) Stores static pages for app and docs (/docs).
Backend Lambda Single modular monolith for auth, workspace, project, files, licenses, access, team, loader, websocket.
Data DynamoDB Multi-table persistence with GSIs for fast lookups.
Object Content S3 Project/file content objects and bundles.
Observability CloudWatch Lambda logs, metrics, alarms, dashboard widgets.

Request Paths

  • /, /login, /register, /dashboard, /workspace/*, /docs/* -> static assets on S3.
  • /api/* -> Lambda Function URL origin.
  • /files/* -> Lambda Function URL origin.

Token and Security Model

  • Auth token format: v2.<payload>.<sig> (HMAC signed).
  • Password hashing: PBKDF2-SHA256 with iteration metadata.
  • Workspace PIN gate via X-Pin-Token for protected operations.
  • Route-level role checks for owner/admin/editor/viewer.

Realtime

WebSocket delivery is supported via API Gateway WebSocket API and Lambda integration. Dashboard/workspace clients consume route-based channels and apply reconnect logic.

See WebSocket Appendix for channel query formats and event examples.
