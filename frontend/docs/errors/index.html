Error Catalog

Most API errors follow a shared JSON structure: {"success": false, "error": "message"}

HTTP Status Guide

Status Typical Meaning Client Action
400 Invalid payload, missing required fields, or malformed parameters. Fix request data and retry.
401 Invalid or expired auth token. Clear auth state and redirect to login.
403 Permission denied or missing PIN verification token. Show role/PIN guidance and stop mutation flow.
404 Entity not found (workspace, project, file, license, invitation). Refresh current listing and show not-found UX.
405 Method not allowed for route. Fix method usage in client/integration.
429 Rate limit exceeded on protected endpoints. Backoff retry and alert user.
500 Unhandled server failure. Show retry option and log context for support.

Common UI Surface Errors

  • Unauthorized deep-link access: workspace route redirects to login with returnTo.
  • PIN required: workspace views lock until pin verification succeeds.
  • Network instability: dashboard/workspace network banners appear on failed fetch or server errors.

Never expose raw secrets in logs or screenshots. Use docs capture masking to replace token/license/webhook values.
